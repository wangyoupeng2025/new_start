# C++ 继承与多态学习笔记

## 一、构造与析构顺序

当一个子类被创建时，先调用父类的构造函数，再调用子类的构造函数。使用结束被销毁时，先调用子类的析构函数，再调用父类的析构函数。

```cpp
class Person {
public:
    Person() { cout << "Person构造" << endl; }
    virtual ~Person() { cout << "Person析构" << endl; }
};

class Student : public Person {
public:
    Student() { cout << "Student构造" << endl; }
    ~Student() { cout << "Student析构" << endl; }
};
二、虚函数与纯虚函数

虚函数使用 virtual 关键字，表明可以被子类重写的函数。子类重写时建议使用 override 关键字。

cpp
virtual void functionName() { }                    // 虚函数
virtual void functionName() override { }           // 子类重写
纯虚函数语法为 virtual void functionName() = 0;。拥有纯虚函数的类称为抽象类，抽象类无法被实例化。抽象类由于会被多态使用，析构函数必须被 virtual 修饰。

cpp
class Shape {
public:
    virtual double area() = 0;  // 纯虚函数
    virtual ~Shape() {}          // 虚析构函数
};
// Shape s;           // ❌ 错误，不能实例化
// Shape* p = new Circle(); // ✅ 正确
注意：所有构造函数都不能被 virtual 修饰。

## 一、构造与析构顺序

当一个子类被创建时，先调用父类的构造函数，再调用子类的构造函数。使用结束被销毁时，先调用子类的析构函数，再调用父类的析构函数。

```cpp
class Person {
public:
    Person() { cout << "Person构造" << endl; }
    virtual ~Person() { cout << "Person析构" << endl; }
};

class Student : public Person {
public:
    Student() { cout << "Student构造" << endl; }
    ~Student() { cout << "Student析构" << endl; }
};
二、虚函数与纯虚函数

虚函数使用 virtual 关键字，表明可以被子类重写的函数。子类重写时建议使用 override 关键字。纯虚函数语法为 virtual void functionName() = 0;。拥有纯虚函数的类称为抽象类，抽象类无法被实例化。抽象类由于会被多态使用，析构函数必须被 virtual 修饰。注意：所有构造函数都不能被 virtual 修饰。

cpp
virtual void functionName() { }                    // 虚函数
virtual void functionName() override { }           // 子类重写
virtual void functionName() = 0;                   // 纯虚函数

class Shape {
public:
    virtual double area() = 0;
    virtual ~Shape() {}
};
// Shape s;              // ❌ 错误，不能实例化
// Shape* p = new Circle(); // ✅ 正确
三、继承中的访问控制

子类会将父类的三个属性全部继承，但 private 属性不能被子类直接访问和使用，需要借助父类中相关的 public 或 protected 函数来访问。

cpp
class Parent {
public:
    int pub = 1;      // 公有：任何地方可访问
protected:
    int pro = 2;      // 保护：子类可访问，外部不可
private:
    int pri = 3;      // 私有：仅本类可访问，子类不可直接访问
};

class Child : public Parent {
public:
    void test() {
        cout << pub << endl;  // ✅ 可以
        cout << pro << endl;  // ✅ 可以
        // cout << pri << endl;  // ❌ 不可以
    }
};
四、父类指针与子类对象

使用父类的指针来创建子类对象：Parent* p = new Child();

为什么用父类指针创建子类对象？

统一存储：当需要存储多种子类时，只需一个 vector<Base*> 容器即可，不需要为每个子类单独创建容器。
统一函数参数：用父类指针作为参数，一个函数可以处理所有子类类型，不需要为每个子类重载函数。
注意事项：父类指针只能访问父类和子类共有的部分，无法直接访问子类独有的方法。

cpp
Animal* p = new Dog();
p->speak();      // ✅ 父类有的方法，可以调用
// p->fetch();   // ❌ 子类独有的方法，不能直接调用
五、向下转型（dynamic_cast）

当需要访问子类独有的方法时，需要将父类指针向下转型为子类指针。

cpp
Dog* dogPtr = dynamic_cast<Dog*>(p);
if (dogPtr != nullptr) {      // 转型成功检查
    dogPtr->fetch();          // 现在可以调用子类独有方法
}
使用条件：父类必须有虚函数（至少要有虚析构函数），会在运行时进行类型检查。

六、虚析构函数

当使用父类指针创建子类对象时，如果父类的析构函数没有被 virtual 修饰，销毁时只会调用父类的析构函数，而不会调用子类的析构函数，导致子类部分的内存泄漏。

cpp
class Parent {
public:
    ~Parent() { }              // ❌ 不是虚函数，子类析构不会被调用
};

class Parent {
public:
    virtual ~Parent() { }      // ✅ 虚析构函数，正确调用子类析构
};
七、对象切片

什么是切片？ 当把子类对象赋值给父类对象时，子类仅保留从父类继承来的属性和方法，独有的属性和方法全部丢失。

cpp
Dog dog;
dog.name = "旺财";
dog.breed = "金毛";    // 子类独有属性

Animal animal = dog;   // 切片发生！breed 丢失
如何避免切片？ 使用指针或引用。

cpp
Animal* p = &dog;      // ✅ 指针，不切片
Animal& r = dog;       // ✅ 引用，不切片
vector<Animal*> v;     // ✅ 容器存指针，不切片
八、内存管理

内存分区：

栈区：局部变量，函数结束自动释放
堆区：用 new 动态创建的变量，需要手动 delete
全局/静态区：全局变量和被 static 修饰的局部变量
指针变量在栈区（除非指针本身也是静态或全局的），被指针指向的变量空间在堆区。

实例化的两种方式：

cpp
// 方式一：栈对象
Student s1("小明", 18, "123456", "S001");
// s1 对象存放在栈区，自动管理生命周期

// 方式二：堆对象（指针方式）
Student* s2 = new Student("小红", 19, "789012", "S002");
// 指针 s2 存放在栈区，s2 所指向的对象存放在堆区
// 需要手动 delete s2
选择建议：

大部分情况下使用栈对象（方式一），简单安全
当要把不同子类存放到同一个容器中时，必须使用指针方式（方式二），因为栈对象存储会发生切片
现代 C++ 推荐使用智能指针：auto s3 = make_unique<Student>("小华", 20, "345678", "S003");
