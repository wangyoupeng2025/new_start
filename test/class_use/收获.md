# C++ 学习笔记：引用、运算符重载、深拷贝与析构函数

## 1. 引用（Reference）

### 基本概念
- `&` 是引用的符号，相当于给变量起**别名**
- 引用的是变量的**地址**
- 引用必须是**已经存在**的内容

### 示例代码
int a = 8;
int &b = a;    // b 是 a 的引用（别名）
b = 10;        // 此时 a 的内容被改为 10，证明是地址传递

### 底层原理
> `b` 相当于一个指针，指向 `a` 的空间（但语法上作为别名使用）

## 2. 运算符重载

### 基本语法
返回类型 operator 要重载的运算符(传入的参数) {
    // 重载的内容
}

### 重载 `<<` 运算符示例
ostream& operator<<(ostream& os, Sex s) {
    os << (s == male ? "男" : "女");
    return os;
}

### 代码解释
- `os` 是输出流（如 `cout`）
- `<<` 是输出运算符
- 意思：把后面的内容输出到 `os`

### 为什么返回 `ostream&` 而不是 `ostream`？

| 返回类型 | 效果 | 问题 |
|---------|------|------|
| ostream&（引用） | 返回对象本身 | ✅ 可以链式调用 |
| ostream（值） | 返回对象的副本 | ❌ 副本在表达式结束后销毁 |

### 链式调用的执行过程
cout << sex << "你好" << endl;

第1步：cout << sex
  ├─ 调用 operator<<(cout, sex)
  ├─ 输出 "男"
  └─ 返回 cout（重要！）

第2步：(cout) << "你好"
  ├─ cout << "你好"（内置功能）
  ├─ 输出 "你好"
  └─ 返回 cout

第3步：(cout) << endl
  ├─ cout << endl（换行）
  └─ 返回 cout

### 关于副本的说明
cout << sex << "你好";

实际过程：
1. operator<< 返回副本（临时对象）
2. 临时对象 << "你好" ← 这个操作是合法的
3. 整个表达式结束后，临时对象才销毁

问题：输出到了临时对象，不是真正的 cout，所以看不到结果

结论：副本会在整个表达式结束后被销毁，
     但输出到副本的内容不会显示在屏幕上，
     因为副本不是真正的 cout。

## 3. 析构函数

### 基本语法
~类名() {
    // 释放动态开辟的空间
}

### 特点
- 没有参数
- 没有返回值
- 职责：释放此类中动态开辟的空间，防止内存泄漏

### ⚠️ 重要陷阱
// ❌ 错误："John" 是字符串常量，不能手动释放
char* p = "John";
delete p;  // 错误！

// ✅ 正确：用 new 开辟的才能用 delete 释放
name = new char[strlen(n) + 1];  // 分配内存
strcpy(name, n);                  // 复制内容
// ... 使用完毕后
delete[] name;                    // 释放内存

## 4. 浅拷贝 vs 深拷贝

### 浅拷贝（默认）
student(const student& other) {
    name = other.name;     // 只是复制地址
    address = other.address; // 只是复制地址
}
- 对应内容的指针直接指向原内容
- **不需要**额外开辟空间

### 深拷贝
student(const student& other) {
    name = new char[strlen(other.name) + 1];
    strcpy(name, other.name);
    
    age = other.age;
    sex = other.sex;
    
    address = new char[strlen(other.address) + 1];
    strcpy(address, other.address);
}
- 先开辟新空间，再进行赋值操作
- **需要**额外开辟空间
- **需要**析构函数释放

### 为什么存入 vector 时调用深拷贝？
> 因为创建 student 对象时传入的是 `st1`，而不是分散的属性值

## 5. 拷贝赋值运算符

### 完整实现
student& operator=(const student& other) {
    if (this != &other) {  // 防止自赋值
        delete[] name;      // 释放旧内存
        delete[] address;
        
        name = new char[strlen(other.name) + 1];
        strcpy(name, other.name);
        age = other.age;
        sex = other.sex;
        address = new char[strlen(other.address) + 1];
        strcpy(address, other.address);
    }
    return *this;
}

### 使用场景

| 场景 | 说明 | 操作 |
|------|------|------|
| st2 = st1 | 赋值给已存在的对象 | 先 delete 释放旧内存，再深拷贝 |
| st1 = st1 | 自己赋值自己 | this == &other，跳过操作，直接返回 |

### 为什么要先 delete？
> 被赋值的对象要先释放自己原先属性动态开辟的内存，再重新开辟空间进行深拷贝

## 6. const 在拷贝构造函数中的作用

const student& other

- 可以接收 const 对象
- 保证不会修改 other
- 这是拷贝构造函数的**标准写法**

## 7. 三法则（Rule of Three）

> 如果类需要**析构函数**，通常也需要：

1. **拷贝构造函数**
2. **拷贝赋值运算符**
3. **析构函数**

## 9. vector 与拷贝的总结

以 vector 为例：

vector 在存储对象时会调用拷贝构造函数。
- 如果实现了深拷贝 → 深拷贝
- 如果使用默认拷贝构造函数 → 浅拷贝

浅拷贝的问题：
  容器指针指向原数据地址 → 多次释放同一空间 → 程序崩溃

深拷贝的解决：
  重新开辟空间 → 复制值 → 各自释放各自指向的地址

## 📌 核心要点速记

- **引用**：别名，地址传递，必须存在
- **运算符重载返回引用**：支持链式调用
- **析构函数**：释放动态内存，delete[] 配 new[]
- **深拷贝**：开辟新空间，复制内容
- **三法则**：析构 + 拷贝构造 + 拷贝赋值
