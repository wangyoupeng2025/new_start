# C++ 初始化列表与智能指针学习笔记

## 一、初始化列表（Initializer List）

### 1. 基本语法

初始化列表是构造函数中在函数体执行之前初始化成员变量的语法，使用冒号 `:` 开头。

```cpp
Student(const char* name, int age, string phonenumber, string Xuehao)
    : Person(name, age, phonenumber) {
    cout << "这是Student类的构造" << endl;
    this->Xuehao = Xuehao;
}
2. 为什么必须使用初始化列表调用父类构造？

执行子类构造函数时，会先调用父类的构造函数。即使写了父类的有参构造，如果没有显式调用，编译器还是会调用父类默认的无参构造。

但是，一旦写了父类的有参构造，默认无参构造就会被废弃，导致父类失去无参构造。

子类的 name、age 等属性来自父类，必须通过父类的初始化才能被子类继承。编译器会默认调用父类无参构造去初始化，即在进入 Student(...){...} 之前会调用 Person(){}。但无参构造已经废弃，找不到这个构造函数，编译器就会报错。

解决方法： 使用 Person(name, age, phonenumber) 显式调用父类的有参构造来初始化这些属性，子类才能正常继承。

核心原则：继承来的属性必须由父类初始化，子类只是继承而来。

3. 初始化列表 vs 函数体赋值

cpp
// 方式1：初始化列表（推荐）
Student(string n, int a) : name(n), age(a) {
    // 直接初始化
    // 过程：
    // 1. name 直接用 n 构造（一步到位）
    // 2. age 直接用 a 构造（一步到位）
}

// 方式2：函数体赋值（不推荐）
Student(string n, int a) {
    name = n;  // 过程：name 先默认构造为空字符串，再调用赋值运算符 = n
    age = a;   // 过程：age 先被默认初始化为随机值，再被赋值为 a
}
推荐使用初始化列表，可以减少不必要的构造步骤，提高效率。

4. 必须使用初始化列表的情况

情况1：const 成员

cpp
class Demo {
private:
    const int id;  // const 成员一旦初始化就不能修改

public:
    // ❌ 错误：const 成员不能在函数体内赋值
    Demo(int x) {
        id = x;  // 编译错误！
    }

    // ✅ 正确：必须用初始化列表
    Demo(int x) : id(x) {
    }
};
情况2：引用成员

cpp
class Demo {
private:
    int& ref;  // 引用成员

public:
    // ❌ 错误：引用必须在创建时绑定
    Demo(int& r) {
        ref = r;  // 编译错误！
    }

    // ✅ 正确：必须用初始化列表
    Demo(int& r) : ref(r) {
    }
};
情况3：成员对象没有无参构造

cpp
class Inner {
public:
    Inner(int x) { }  // 只有有参构造，没有默认构造
};

class Outer {
private:
    Inner obj;  // Inner 没有默认构造

public:
    // ❌ 错误：obj 需要默认构造，但没有
    Outer() {
        // 这里才想初始化 obj，但已经晚了
    }

    // ✅ 正确：必须在初始化列表调用有参构造
    Outer() : obj(10) {
    }
};
情况4：父类没有默认构造

cpp
class Person {
public:
    Person(string name, int age) { }  // 只有有参构造
};

class Student : public Person {
public:
    // ❌ 错误：Person 需要默认构造，但没有
    Student(string name, int age, string id) {
        // Person 部分没有被初始化
    }

    // ✅ 正确：必须用初始化列表调用父类构造
    Student(string name, int age, string id)
        : Person(name, age) {  // 先初始化父类部分
        this->id = id;
    }
};
二、智能指针（Smart Pointers）

1. 三种智能指针对比

指针类型	所有权	使用场景	开销
unique_ptr	独占	明确只有一个所有者	极低（零开销）
shared_ptr	共享	多个所有者	较高（引用计数）
weak_ptr	弱引用	配合 shared_ptr 打破循环引用	较低
2. 基本使用格式

一般用于创建类对象时使用：

cpp
auto student = make_unique<Student>(...);
auto teacher = make_shared<Teacher>(...);
3. weak_ptr 解决循环引用

问题代码（内存泄漏）

cpp
class Person {
public:
    shared_ptr<Person> friend;
};

auto p1 = make_shared<Person>();
auto p2 = make_shared<Person>();
p1->friend = p2;
p2->friend = p1;
// p1 和 p2 互相引用，永远不会释放！内存泄漏！
解决方案（使用 weak_ptr）

cpp
class Person {
public:
    weak_ptr<Person> friend;  // 改用 weak_ptr
};

auto p1 = make_shared<Person>();
auto p2 = make_shared<Person>();
p1->friend = p2;
p2->friend = p1;
// 可以正常释放 ✅
