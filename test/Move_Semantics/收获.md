# 移动语义学习笔记

## 一、左值和右值的概念

### 1. 核心定义

| 概念 | 定义 | 判断方法 |
|------|------|---------|
| 左值 | 有内存地址、有名字、可以取地址 | `&x` 能成功 |
| 右值 | 临时值、没名字、不能取地址 | `&10` 会报错 |

```cpp
int x = 10;      // x 是左值（有地址）
                 // 10 是右值（临时值，没地址）

string s = "hello";  // s 是左值
                     // "hello" 是右值
2. 直观理解

text
左值 = 左边可以放的值
     = 可以出现在赋值号左边
     = 有持久生命周期的对象

右值 = 右边可以放的值
     = 只能出现在赋值号右边
     = 临时对象，表达式结束后就销毁
cpp
int a = 10;   // a 是左值，10 是右值
int b = a;    // b 是左值，a 是左值（但可以被读取）
int c = a + b; // a+b 是右值（临时结果）
3. 常见例子

表达式	是左值吗	是右值吗	说明
int x = 5; 中的 x	✅	❌	有名字的变量
5	❌	✅	字面常量
x + y	❌	✅	表达式的结果是临时值
&x	❌	✅	取地址的结果是临时指针
"hello"	❌	✅	字符串字面量
std::move(x)	❌	✅	转换成右值引用
4. 判断技巧：能不能取地址？

cpp
int x = 10;
int* p1 = &x;     // ✅ 可以，x 是左值
int* p2 = &10;    // ❌ 错误，10 是右值，不能取地址

string s = "hello";
string* p3 = &s;  // ✅ 可以，s 是左值
string* p4 = &string("temp"); // ❌ 错误，临时对象是右值
二、引用的概念

1. 本质理解

引用就是给变量起一个别名，操作引用就是操作原变量本身，不需要像值传递那样创建副本。

cpp
int x = 10;
int& ref = x;   // ref 是 x 的别名

ref = 20;       // 修改 ref 就是修改 x
cout << x;      // 输出 20
2. 引用 vs 指针

对比	引用 &	指针 *
是什么	别名	存地址的变量
能否为空	❌ 不能	✅ 能（nullptr）
能否改变指向	❌ 不能	✅ 能
语法	直接用，像普通变量	要用 * 解引用
注意： 引用不能当指针用：

引用不需要 & 取地址
引用不需要 * 解引用
引用直接用，就像原变量
三、左值引用和右值引用

核心理解

类型	写法	含义
左值引用	int&	给左值起别名，实际操作左值本身
右值引用	int&&	给右值（临时变量）起名字，实际操作临时变量本身
代码示例

cpp
int x = 10;
int& lref = x;       // ✅ 左值引用绑定左值
int&& rref = 20;     // ✅ 右值引用绑定右值

// int& lref2 = 20;  // ❌ 左值引用不能绑定右值
// int&& rref2 = x;  // ❌ 右值引用不能绑定左值
四、移动语义的核心

std::move 的作用

std::move 的本质是类型转换：把左值变成右值引用。

cpp
vector<string> x;
x.push_back("hello");

vector<string> y = std::move(x);
执行过程：

text
1. std::move(x) 并不移动任何东西
   → 只是把 x 的类型转换成 vector<string>&&（右值引用）

2. 执行 vector 的移动构造函数
   → 移动构造函数被调用，参数是 std::move(x)

3. 移动构造函数内部：
   → y.指针 = x.指针    （直接拿过来）
   → x.指针 = nullptr    （源对象置空）

4. 结果：
   → x 为空（没有资源了）
   → y 拥有资源 "hello"
重要理解

概念	正确理解
std::move 的作用	只是类型转换（左值 → 右值引用），不是移动
谁完成移动	移动构造函数 / 移动赋值运算符
移动后的源对象	有效但为空（可以继续使用，可以析构）
五、为什么需要移动语义？

问题：拷贝开销太大

cpp
vector<int> v(1000000, 1);  // 100 万个 int

vector<int> v2 = v;   // 拷贝：要复制 100 万次！慢！
vector<int> v3 = std::move(v);  // 移动：只转移指针，O(1)！快！
操作	时间复杂度	说明
拷贝	O(n)	复制所有元素
移动	O(1)	只转移内部指针
哪些类型受益最大？

类型	拷贝开销	移动开销	收益
int	小	一样	无
string	大（堆内存）	小（只转移指针）	巨大
vector	大（堆内存）	小（只转移指针）	巨大
map	大	小	巨大
六、编译器自动移动的场景

✅ 会自动移动

场景	示例
返回局部变量	return local;
参数传入临时对象	func(vector<int>(1000));
throw 异常对象	throw result;
❌ 不会自动移动

场景	原因
返回全局/成员变量	可能被其他地方使用
参数传入具名变量	后面可能还用
赋值操作	需要手动 std::move
cpp
// ❌ 不会自动移动
vector<int> v1 = {1, 2, 3};
vector<int> v2 = v1;   // 拷贝（v1 后面还要用）

// ✅ 需要手动
vector<int> v3 = std::move(v1);  // 移动
最佳实践

cpp
// ✅ 依赖自动移动
vector<int> create() {
    vector<int> v(1000000);
    return v;  // 自动移动，不需要写 std::move
}

// ✅ 显式移动
vector<int> v1 = create();
vector<int> v2 = std::move(v1);

// ❌ 不要画蛇添足
vector<int> create() {
    vector<int> v(1000000);
    return std::move(v);  // 阻止返回值优化，反而变慢
}
七、补充：移动后源对象的状态

重要规则

移动后的源对象处于有效但未指定的状态：

可以正常析构
可以重新赋值
不应该依赖它的具体值
cpp
vector<int> v1 = {1, 2, 3};
vector<int> v2 = std::move(v1);

// v1 现在是空，但可以继续使用
v1.push_back(10);   // ✅ 可以
cout << v1.size();  // 1
八、移动语义 vs 拷贝语义对比

对比	拷贝	移动
适用场景	还需要用原对象	原对象不再需要
开销	O(n)	O(1)
源对象状态	不变	有效但空
语法	v2 = v1	v2 = std::move(v1)
九、一句话总结

text
左值 = 有地址的变量
右值 = 临时值，不能取地址

左值引用(&) = 给左值起别名
右值引用(&&) = 给右值起名字

std::move = 类型转换（左值 → 右值引用），不是真正的移动
移动构造函数 = 真正执行资源转移的地方

移动语义 = 转移资源所有权，不复制，O(1) 高效
