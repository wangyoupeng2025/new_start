# C/C++ 宏定义学习笔记

## 1. 基本宏定义
永远不要在宏参数中使用带有副作用的表达式（i++、++i、函数调用等）。
```c
#define PI 3.14
编译时执行文本替换，将所有的 PI 替换成 3.14
宏名通常使用大写字母，以区分变量和函数
2. 带参数的宏

c
#define RUL(a, b) ((a) * (b))
\ 是换行符，用于将宏定义延续到下一行
必须给每个参数加括号 (a) (b)
整个宏体也必须加括号 ((a) * (b))
为什么要加括号？

错误写法：

c
#define BAD_RUL(a, b) a * b
BAD_RUL(1+2, 2+3)
// 替换为：1+2 * 2+3
// 实际计算：1 + (2*2) + 3 = 8，而不是 3*5=15
正确写法：

c
#define RUL(a, b) ((a) * (b))
RUL(1+2, 2+3)
// 替换为：((1+2) * (2+3))
// 计算结果：3 * 5 = 15

4. 跨平台导出宏（EXPORT）

c
#if defined(_MSC_VER)
    // Visual Studio (Windows)
    #define EXPORT __declspec(dllexport)
#elif defined(__GNUC__) || defined(__clang__)
    // GCC / Clang (Linux/Mac)
    #define EXPORT __attribute__((visibility("default")))
#else
    #define EXPORT
#endif
根据不同的编译器/系统，将 EXPORT 替换成不同的文本
被 EXPORT 修饰的函数可以被外界调用
5. 动态库 vs 静态库

类型	文件后缀	特点
可执行文件	.exe (Windows) / 无后缀 (Linux/Mac)	有 main 函数，双击就能运行
静态库	.lib (Windows) / .a (Linux)	代码的"压缩包"，编译时复制到可执行文件中
动态库	.dll (Windows) / .so (Linux) / .dylib (Mac)	代码的"公共仓库"，运行时临时调用，多个程序共享
动态库的关键特性

被 EXPORT 修饰的函数可以被外界调用
即使项目文件和动态库文件不在同一个目录下，也可以调用
多个程序可以共享同一个动态库，节省内存和磁盘空间
6. 可变参数宏（变参宏）

c
#define DEBUG_PRINT(fmt, ...) \
    printf("[%s:%d] " fmt "\n", __FILE__, __LINE__, ##__VA_ARGS__)

// 使用
DEBUG_PRINT("x = %d, y = %d", x, y);
// 输出: [main.c:42] x = 10, y = 20
关键点解释

符号	含义
fmt	占位符，代表格式字符串（如 "x = %d, y = %d"）
...	表示可变数量的参数
__VA_ARGS__	代表 ... 传入的所有参数
##__VA_ARGS__	当参数为空时，自动删除前面的逗号
__FILE__	预定义宏，自动替换为当前文件名
__LINE__	预定义宏，自动替换为当前行号
为什么 __FILE__ 和 __LINE__ 会变成具体值？

它们是预定义宏，编译器在预处理阶段自动替换
__FILE__ 替换成当前源文件的名字（字符串）
__LINE__ 替换成当前代码的行号（数字）
c
// 源代码
printf("%s:%d\n", __FILE__, __LINE__);

// 预处理后（假设文件 test.c，第 3 行）
printf("%s:%d\n", "test.c", 3);
7. 条件调试宏（分级日志）

c
#define DEBUG_LEVEL 2   // 0=关闭, 1=错误, 2=警告, 3=详情

#if DEBUG_LEVEL >= 1
    #define LOG_ERROR(fmt, ...) printf("[ERROR] " fmt "\n", ##__VA_ARGS__)
#else
    #define LOG_ERROR(fmt, ...)  // 空定义，什么也不做
#endif

#if DEBUG_LEVEL >= 2
    #define LOG_WARN(fmt, ...)  printf("[WARN]  " fmt "\n", ##__VA_ARGS__)
#else
    #define LOG_WARN(fmt, ...)
#endif
通过修改 DEBUG_LEVEL 的值，控制哪些日志被编译
DEBUG_LEVEL = 0：所有日志都被编译为空（不产生任何代码）
DEBUG_LEVEL = 2：LOG_ERROR 和 LOG_WARN 生效

## 11. 多语句宏的 do-while(0) 包装

### 为什么要用 do-while(0)？

**核心原因：让多语句宏在任何上下文中都像一个普通语句一样安全使用。**

### 问题演示：普通多语句宏的陷阱

错误写法：
```c
#define SAFE_FREE(p) free(p); p = NULL;
在 if 语句中使用时出现问题：

c
if (ptr != NULL)
    SAFE_FREE(ptr);
else
    printf("ptr is NULL\n");
展开后：

c
if (ptr != NULL)
    free(ptr); p = NULL;  // p = NULL 不在 if 里面！
else                      // 编译错误：else 没有对应的 if
    printf("ptr is NULL\n");
常见的错误尝试

尝试1：用大括号

c
#define SAFE_FREE(p) { free(p); p = NULL; }
// 展开后：if (...) { ... }; else → 分号导致 else 悬空
尝试2：调用时不加分号

c
#define SAFE_FREE(p) { free(p); p = NULL; }
if (ptr != NULL)
    SAFE_FREE(ptr)   // 不加分号，能用但违反习惯
正确方案：do-while(0)

c
#define SAFE_FREE(p) do { free(p); p = NULL; } while(0)
在 if 中使用：

c
if (ptr != NULL)
    SAFE_FREE(ptr);   // ✅ 分号正常，展开后是 do{...}while(0);
else
    printf("ptr is NULL\n");
展开后：

c
if (ptr != NULL)
    do { free(ptr); ptr = NULL; } while(0);
else
    printf("ptr is NULL\n");
为什么这样写有效？

特性	说明
do-while 是完整语句块	后面必须加分号，符合习惯
while(0) 条件为假	循环体只执行一次
编译器优化	完全移除循环结构，零运行时开销
块级作用域	可以在宏内定义局部变量
常见应用场景

场景1：多条语句的宏

c
#define SWAP(a, b) do { \
    typeof(a) _tmp = (a); \
    (a) = (b); \
    (b) = _tmp; \
} while(0)
场景2：带 return 的宏（在函数中使用）

c
#define CHECK_NULL(ptr) do { \
    if ((ptr) == NULL) { \
        printf("NULL pointer!\n"); \
        return -1; \
    } \
} while(0)

int func(void* p) {
    CHECK_NULL(p);  // ✅ 安全返回
    return 0;
}
场景3：带 break 的宏（在循环中使用）

c
#define CHECK_ERROR(ret) do { \
    if ((ret) < 0) { \
        printf("Error: %d\n", ret); \
        break; \
    } \
} while(0)

while (condition) {
    CHECK_ERROR(result);
    // 出错时 break 跳出循环
}
对比总结

宏写法	在 if 中安全	需要加分号	会被优化
{ ... }	❌ 分号问题	否（矛盾）	-
多条语句	❌ else 悬空	是	-
do{...}while(0)	✅ 安全	✅ 是	✅ 是
一句话总结

do { ... } while(0) 是多语句宏的标准写法，让宏在任何上下文中都像一个普通语句一样工作，且编译后零开销。
## 12. inline 函数

### inline 的核心作用

**inline 告诉编译器：把函数的代码直接复制到调用它的地方，避免函数调用的跳转开销。**

| 作用 | 说明 |
|------|------|
| 消除函数调用开销 | 将函数代码嵌入调用点（省去压栈、跳转、返回） |
| 解决多重定义 | 允许在头文件中定义函数，多个 `.c` 文件包含也不会报错 |

> **注意**：`inline` 只是给编译器的**建议**，编译器可以忽略（如函数太长或递归）。

### 什么时候用 inline？

**✅ 适合 inline 的场景：**

```c
// 1. 非常短小的函数（1-5 行）
static inline int square(int x) {
    return x * x;
}

// 2. 在头文件中定义函数（C 语言必须加 static inline）
// utils.h
#ifndef UTILS_H
#define UTILS_H
static inline int add(int a, int b) {
    return a + b;
}
#endif

// 3. 频繁调用的简单函数
static inline int clamp(int x, int low, int high) {
    return x < low ? low : (x > high ? high : x);
}
❌ 不适合 inline 的场景：

场景	原因
函数体很长（>10 行）	内联会导致代码膨胀
递归函数	编译器无法内联递归
包含循环的函数	循环开销远大于调用开销
虚函数（C++）	多态调用无法内联
调试阶段	内联后无法打断点
C vs C++ 的 inline 写法

语言	头文件中的写法
C 语言	static inline int func() { ... }
C++	inline int func() { ... }
inline vs static vs static inline

写法	含义	使用场景
inline	建议内联，允许多重定义	C++ 头文件
static	每个 .c 有独立副本	文件内部函数
static inline	建议内联 + 内部链接	C 语言头文件推荐
一句话总结

inline = "允许在头文件里定义函数" + "建议编译器把代码复制过去（避免跳转开销）"
13. constexpr（C++11 起）

constexpr 的核心作用

constexpr 让函数或变量在编译期求值，将运行时的计算提前到编译阶段。

cpp
constexpr int square(int x) {
    return x * x;
}

int arr[square(5)];  // 编译期计算：square(5) = 25，数组大小确定
constexpr 的用途

用途	示例
编译期常量	constexpr int MAX_SIZE = 1024;
编译期函数	constexpr int factorial(int n) { return n <= 1 ? 1 : n * factorial(n-1); }
数组大小	int arr[factorial(5)];
模板参数	std::array<int, square(10)> arr;
constexpr 函数的要求

要求	说明
返回值类型	必须是字面类型（int、double、指针等）
参数类型	必须是字面类型
函数体	只能包含一条 return 语句（C++11）；C++14 后可包含多个语句
不能有	循环（C++11）、static 变量、goto、try-catch
C++11 vs C++14 vs C++17 的 constexpr

cpp
// C++11：只能有一条 return 语句
constexpr int square(int x) {
    return x * x;
}

// C++14：可以有多个语句和循环
constexpr int factorial(int n) {
    int result = 1;
    for (int i = 2; i <= n; ++i) {
        result *= i;
    }
    return result;
}

// C++17：可以用于 if constexpr（编译期条件判断）
template<typename T>
auto get_value(T t) {
    if constexpr (std::is_pointer_v<T>) {
        return *t;
    } else {
        return t;
    }
}
constexpr vs inline vs 普通宏

特性	宏	inline	constexpr
类型检查	❌ 无	✅ 有	✅ 有
编译期计算	⚠️ 常量替换	❌ 运行时	✅ 编译期
作用域控制	❌ 全局	✅ 有	✅ 有
调试支持	❌ 困难	✅ 可打断点	✅ 可打断点
多次求值风险	❌ 有	✅ 无	✅ 无
完整示例

cpp
#include <iostream>
#include <array>

// 编译期阶乘
constexpr int factorial(int n) {
    int result = 1;
    for (int i = 2; i <= n; ++i) {
        result *= i;
    }
    return result;
}

// 编译期平方
constexpr int square(int x) {
    return x * x;
}

int main() {
    // 编译期计算
    constexpr int fact5 = factorial(5);  // 120
    constexpr int sq10 = square(10);     // 100
    
    // 用作数组大小
    int arr[factorial(4)];  // 24 个元素
    
    // 用作模板参数
    std::array<int, square(6)> arr2;  // 36 个元素
    
    // 运行时调用也可以（不要求编译期）
    int x = 3;
    int y = square(x);  // 运行时计算也可以
    
    std::cout << "5! = " << fact5 << std::endl;
    std::cout << "10^2 = " << sq10 << std::endl;
    
    return 0;
}
什么时候用 constexpr？

场景	是否使用	说明
数组大小需要常量	✅ 使用	int arr[compute_size()]
模板参数	✅ 使用	std::array<int, N>
数学常量函数	✅ 使用	constexpr double pi() { return 3.14159; }
简单的 getter	✅ 使用	constexpr int getX() const { return x; }
复杂的 I/O 操作	❌ 不用	编译期无法进行文件读写
动态内存分配	❌ 不用（C++14 前）	C++14 后有限支持
constexpr 与 const 的区别

特性	const	constexpr
含义	值不可修改	编译期已知的值
初始化	运行时或编译期	必须编译期
函数	不能修饰函数	可以修饰函数
变量	const int x = getValue();	constexpr int x = 5;
一句话总结

constexpr = "让编译器在编译阶段就把这个值算出来"——提前计算，运行时零开销。
14. inline 与 constexpr 的关系（C++）

C++11 起，constexpr 隐含 inline

cpp
// 这两种写法等价
constexpr int square(int x) { return x * x; }

// 编译器会自动把它当作 inline 处理
对比总结

特性	inline	constexpr
适用语言	C / C++	C++11 起
核心作用	消除调用开销 + 允许多重定义	编译期求值
是否隐含 inline	不适用	✅ 是（C++11 起）
能否用于变量	❌	✅
能否用于函数	✅	✅
选择建议

text
需要头文件里定义短函数 → 用 inline（C）或直接写（C++）
需要编译期计算常量 → 用 constexpr（C++）
两者都需要 → 用 constexpr（自动隐含 inline）
